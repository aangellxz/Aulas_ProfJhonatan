Catálogo da Loja de Games

Você foi contratado para construir o núcleo de dados de um catálogo. Sem telas e sem internet — tudo roda no console. O que importa é modelar bem os dados com objetos.

Unidade Curricular Desenvolvimento de Sistemas Módulo 4 — JavaScript Stack JavaScript puro (console) Entrega individual ou em dupla

PROJETO: Loja de Games Pixel Play

A Pixel Play é uma loja de games que está informatizando o catálogo. Antes de construir telas, você precisa entregar o núcleo de dados: cada jogo do catálogo precisa virar um objeto que carrega suas próprias informações e sabe se descrever e decidir sozinho (usando this).

Em cima desses objetos, o gerente quer alguns relatórios rápidos no console: o que está disponível, quantas unidades existem no estoque e quantos jogos há de cada gênero. Trate o código como um sistema, não como um rascunho de variáveis soltas.

Objetivo

Construir, em JavaScript puro, um catálogo de jogos em que cada jogo é um objeto com propriedades e métodos que usam this, e gerar relatórios percorrendo e transformando a lista. Todos os itens abaixo são obrigatórios e serão verificados na entrega:

objeto (chave: valor) propriedades acesso ponto e colchete método + this array de objetos forEach map / filter / reduce for..in

O que o programa precisa ter

  • Pelo menos 3 jogos, cada um como objeto, guardados em um array (catalogo).
  • Cada jogo com, no mínimo: titulo, preco, genero, unidades, lancado e plataformas.
  • Métodos de objeto que usam this para falar dos próprios dados.
  • Relatórios no console: listagem numerada, jogos por gênero e totais — sempre percorrendo ou transformando a lista.
  • Código organizado em funções pequenas, sem var.

O formato de cada jogo

Cada jogo do catálogo é um objeto com estas propriedades. Você define os valores; os métodos você cria ao longo das etapas.

const jogo = {
  titulo: "...",        // texto
  preco: 0,             // número
  genero: "...",        // texto
  unidades: 0,          // número (quantos há em estoque)
  lancado: true,        // booleano (já está à venda?)
  plataformas: ["..."]  // lista (array) de textos
  // + os métodos da Etapa 2 entram aqui dentro
};

Etapas do desenvolvimento

1

Criar o objeto e acessar propriedades

Crie um objeto chamado jogo com as 6 propriedades do formato acima. A propriedade plataformas é uma lista de textos (ex.: ["PC", "PS5", "Xbox"]).

Depois imprima, nesta ordem: o titulo usando ponto, o genero usando colchete, e a primeira plataforma da lista.

Exemplo de saída (se o jogo for o Elden Ring)

Elden Ring
RPG
PC
objeto propriedades acesso ponto e colchete
2

Dar comportamento ao objeto (métodos e this)

Adicione dois métodos ao seu jogo. Os dois leem os dados só por this.

Método 1 — statusEstoque() (sem parâmetro): olha this.unidades e decide o que devolver:

  • 0 → devolve "Esgotado"
  • de 1 a 3 → devolve "Últimas unidades"
  • 4 ou mais → devolve "Disponível"

Método 2 — cabeNoOrcamento(valor) (com parâmetro): devolve true se this.preco for menor ou igual a valor, senão false.

Exemplo de saída (com unidades: 5 e preco: 199.9)

Disponível
true
false
método + this parâmetro return
3

Montar a lista e mexer nas pontas

Crie 3 jogos completos (mesmo formato, todos com os dois métodos da Etapa 2). Varie genero, preco e unidades entre eles. Monte o array catalogo com esses 3 jogos.

Depois aplique nesta ordem:

  • um jogo novo chega e entra no fim da lista;
  • o jogo que está no começo saiu de linha — remova-o;
  • um lançamento em destaque chega e entra no começo.

No fim, imprima quantos jogos sobraram (catalogo.length).

array de objetos push shift unshift .length
4

Listar o catálogo

Crie a função listarCatalogo(jogos) que percorre o catalogo com forEach e imprime cada jogo numerado a partir de 1, mostrando o status de estoque (use o método statusEstoque() da Etapa 2).

O forEach entrega o item e o índice — use o índice para numerar (indice + 1).

Formato da saída

1 - Elden Ring (Disponível)
2 - Hades (Últimas unidades)
3 - Forza Horizon (Esgotado)
forEach índice método do objeto
5

Relatórios com filter, map e reduce

Sobre o catalogo, gere três relatórios, cada um em sua própria variável:

  • filter + map: filtre só os jogos de um gênero (ex.: "RPG") e transforme esse resultado numa lista só com os titulo. Ex.: ["Elden Ring", "Dark Souls"]
  • reduce de soma: some o total de unidades em estoque de todos os jogos.
  • reduce de comparação: descubra o jogo mais caro e imprima o titulo dele.
filter map reduce
6

Desafio: contar jogos por gênero

Descubra quantos jogos há por gênero — sem saber os gêneros de antemão.

  • Crie um objeto vazio contagemGenero = {}.
  • Percorra o catalogo; para cada jogo, some 1 na chave do gênero dele. Se a chave ainda não existir, comece em 0 antes de somar. O acesso é por colchete (contagemGenero[j.genero]), porque a chave vem de uma variável.
  • Percorra contagemGenero com for..in e imprima cada gênero com a contagem.

Formato da saída

RPG: 2
Roguelike: 1
Corrida: 1
objeto como contador chave dinâmica [ ] for..in

Checklist de entrega

  • Objeto jogo com propriedade que é um array (plataformas) e acesso a um item dela.
  • Leitura de propriedade por ponto e por colchete.
  • Método de decisão (statusEstoque) e método que devolve true/false (cabeNoOrcamento), ambos com this.
  • Array catalogo manipulado nas duas pontas + remoção (push, shift, unshift) e uso de .length.
  • Listagem numerada com forEach + índice.
  • Relatórios com filter, map e reduce (um de cada).
  • Objeto contador por gênero com chave dinâmica + for..in.
  • Código sem var, organizado em funções pequenas.

Regras importantes

  • Use let/const, nunca var.
  • Cada jogo é um objeto literal { } — nada de class (POO vem depois).
  • Os métodos leem os dados só via this — o objeto se vira sozinho, não depende de variável de fora.
  • Cada função tem uma responsabilidade e devolve o resultado com return (o console.log fica com quem chama).
  • Fase JavaScript puro no console: sem fetch, JSON, localStorage ou desestruturação.
